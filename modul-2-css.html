Lektion 1: Introduktion till CSS — syntax och selektorer

CSS-spaning — STEG 1: Den här sektionen använder selektorer på riktigt. Vi har en kombinerad selektor (.lektion-1 p) — en class selector kombinerad med en type selector (descendant selector) — som ger alla paragrafer extra mellanrum. En class selector (.highlight-selektor) som markerar nyckelord med en röd understrykning. En id selector (#css-regel-demo) som ger en unik ruta en röd vänsterkant. Och en group selector (.lektion-1 h2, .lektion-1 h3) som ger båda rubriknivåerna extra teckenavstånd. Allt det du lär dig i den här lektionen — i aktion!

CSS = Cascading Style Sheets. Det är språket som bestämmer hur HTML-element ser ut. Utan CSS ser alla webbsidor ut som tråkiga textdokument från 1995. Med CSS kan du göra vad som helst — färger, layout, animationer, allt.

Du har redan sett CSS i aktion genom hela Modul 1. Nu ska du lära dig skriva det.

Tre sätt att lägga till CSS

MetodHurNär?
Inline <p style="color: red;"> Snabba tester. Använd inte i riktiga projekt.
Intern (style-tagg) <style> i <head> Enkla sidor, övningar. Den här presentationen använder det!
Extern (.css-fil) <link rel="stylesheet" href="style.css"> Alltid i riktiga projekt. Bästa sättet.
Varför extern? Med en extern CSS-fil kan du styla flera HTML-sidor med samma fil. Ändra en rad CSS — och hela webbplatsen uppdateras. Den här presentationen använder en intern <style>-tagg (metod 2) för att allt ska vara i en fil, men i dina egna projekt ska du använda en extern .css-fil.

CSS-syntax — en regel i taget

All CSS bygger på samma mönster:

selektor {
  egenskap: värde;
  egenskap: värde;
}

Exempel — gör alla paragrafer blåa med större text:

p {
  color: blue;
  font-size: 18px;
}

Selektorn bestämmer vilka element som påverkas. Egenskaperna bestämmer vad som ändras. Glöm inte semikolon efter varje rad!

Selektorer — vem ska stylas?

TypSyntaxExempelVad den väljer
Type taggnamn p { ... } Alla <p>-element på sidan.
Class .klassnamn .intro { ... } Alla element med class="intro".
ID #idnamn #header { ... } Det ENE elementet med id="header".
Grupp a, b h1, h2 { ... } Alla <h1> OCH alla <h2>.
Universell * * { ... } ALLA element. Använd sparsamt.

Class vs ID — vad är skillnaden?

Tänk så här: en class är som en t-shirt — flera personer kan ha samma. Ett id är som ett personnummer — bara EN person har det.

<!-- Class: kan användas på flera element -->
<p class="viktigt">Första viktiga texten.</p>
<p class="viktigt">Andra viktiga texten.</p>

<!-- ID: unikt — bara ETT element -->
<h1 id="huvudrubrik">Välkommen!</h1>

<!-- CSS -->
.viktigt {
  color: red;
  font-weight: bold;
}

#huvudrubrik {
  font-size: 3em;
  text-align: center;
}

Uppdrag: Selektorer

  1. Skapa en ny HTML-fil med en intern <style>-tagg.
  2. Lägg till tre paragrafer. Ge två av dem class "highlight" och en av dem id "special".
  3. Skriv CSS som gör: alla <p> mörkgrå, alla med class highlight gula, och den med id special röd och fet.
  4. Lägg till en h1 och en h2 och skriv en group selector som ger båda samma typsnitt.

Boss-level: Selektor-jakt

  1. Öppna den här presentationen i webbläsaren.
  2. Högerklicka → Inspektera på olika element.
  3. Hitta minst en type selector, en class selector och en id selector i CSS:en. Skriv ner dem!

Lektion 2: Färger, typsnitt, marginaler och padding

CSS-spaning — STEG 2: Nu har vi lagt till rgba()-färger! Se rutorna nedan — de har halvgenomskinliga bakgrunder: rgba(233, 69, 96, 0.15) (röd, 15% synlig) och rgba(13, 181, 230, 0.12) (blå, 12% synlig). Du kan se den mörka bakgrunden skina igenom! Vi demonstrerar också tre olika font-stacks med serif, sans-serif och monospace. Dessutom har line-height ökats till 1.7 — texten andas lite mer.

Färger i CSS

Det finns flera sätt att ange färger. Alla gör samma sak — de bestämmer en färg — men de skrivs olika.

FormatExempelNär?
Namngivna color: red; Snabba tester. Det finns ~140 färgnamn.
Hex color: #ff0000; Vanligast. 6 tecken: RR GG BB (hexadecimalt).
RGB color: rgb(255, 0, 0); Samma som hex men med decimaltal (0-255).
RGBA color: rgba(255, 0, 0, 0.5); Som RGB men med genomskinlighet! Sista värdet (alpha) = 0 (osynlig) till 1 (helt synlig).

Den här rutan har en rgba-bakgrund! background-color: rgba(233, 69, 96, 0.15) — röd färg med bara 15% synlighet. Bakgrunden skiner igenom. Snyggt, eller hur?

Den här är blå. background-color: rgba(13, 181, 230, 0.12). Samma grej, annan färg. RGBA är perfekt för subtila bakgrunder som inte tar över.

Typsnitt (font)

Tre viktiga saker om typsnitt:

1. Font-stacks — alltid ha en backup. Inte alla datorer har samma typsnitt installerat. Därför listar du flera alternativ:

/* Webbläsaren testar det första typsnittet.
   Finns det inte → nästa i listan. Och så vidare. */

font-family: 'Segoe UI', system-ui, sans-serif;
font-family: Georgia, 'Times New Roman', serif;
font-family: 'Consolas', 'Courier New', monospace;

Den här texten använder Georgia (serif) — med klackar på bokstäverna. Känns lite mer som en bok eller tidning. Vi har också lagt till font-style: italic och line-height: 1.8.

Den här texten använder Consolas (monospace) — varje tecken är lika brett. Perfekt för kod, men konstigt för brödtext.

Den här texten använder Segoe UI (sans-serif) — rent och modernt. Utan klackar. Vanligast på webben idag.

2. Storlek — px, em och rem.

EnhetVad det betyderExempel
px Pixlar. Fast storlek. font-size: 16px;
em Relativt till förälderns fontstorlek (för font-size) eller elementets egen fontstorlek (för padding, margin m.m.). 2em = dubbelt så stort. font-size: 1.2em;
rem Relativt till <html>-elementets storlek. Förutsägbart. font-size: 1rem;

3. Andra font-egenskaper:

p {
  font-family: 'Segoe UI', sans-serif;  /* Typsnitt */
  font-size: 16px;                       /* Storlek */
  font-weight: bold;                     /* Tjocklek: normal, bold, 100-900 */
  font-style: italic;                    /* Stil: normal, italic */
  line-height: 1.6;                      /* Radavstånd — 1.6 = 160% av textstorleken */
}

Margin vs padding — jacka-metaforen

Det här är den viktigaste skillnaden att förstå:

Tänk dig att du har en jacka på dig.
Padding = kuddarna inuti jackan. Mer padding = mer luft mellan dig (innehållet) och jackans ytterkant (ramen).
Margin = avståndet till personen bredvid dig. Mer margin = mer luft mellan din jacka och grannen. 
/* Padding — luft INUTI elementet */
padding: 20px;              /* Samma på alla sidor */
padding: 10px 20px;         /* 10px topp/botten, 20px vänster/höger */
padding-top: 10px;          /* Bara toppen */

/* Margin — avstånd UTANFÖR elementet */
margin: 20px;               /* Samma på alla sidor */
margin: 10px 20px;          /* 10px topp/botten, 20px vänster/höger */
margin: 10px 20px 30px 40px; /* topp, höger, botten, vänster (medurs!) */
margin-bottom: 30px;        /* Bara botten */

Uppdrag: Färg och typsnitt

  3. Skapa tre stycken (<p>) med tre olika font-stacks (serif, sans-serif, monospace).
  4. Ge ett element en rgba()-bakgrund med 50% genomskinlighet.
  5. Testa skillnaden mellan padding: 20px och margin: 20px — inspektera med F12!

Lektion 3: Boxmodellen och enkel layout

CSS-spaning — STEG 3: Den här sektionen har fått box-sizing: border-box på alla element (via .lektion-3 *) och max-width: 860px med margin-left: auto; margin-right: auto — det centrerar hela innehållet! Märker du att texten inte längre sträcker sig hela vägen till kanterna? Jämför med lektion 1 och 2 ovan. Det är enklaste sättet att göra en sida mer läsbar.

Allt i CSS är en låda. Varje element — varje rubrik, varje paragraf, varje bild — är en rektangulär låda. Förstår du boxmodellen, förstår du CSS-layout.

Boxmodellen — fyra lager

Tänk dig en tavla i en ram:

box-sizing: border-box — din nya bästa kompis

Utan border-box händer konstiga saker. Kolla:

/* UTAN border-box (standard) */
div {
  width: 300px;
  padding: 20px;
  border: 10px solid black;
}
/* Totalbredd = 300 + 20 + 20 + 10 + 10 = 360px!!! */

/* MED border-box */
div {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 10px solid black;
}
/* Totalbredd = 300px. Punkt. Padding och border ryms inuti. */

Därför börjar nästan alla moderna projekt med den här regeln:

* {
  box-sizing: border-box;
}
/* Stjärnan (*) = universell selektor = ALLA element */

display — hur element beter sig

Varje HTML-element har ett display-värde som bestämmer hur det placeras:

VärdeBeteendeExempel-element
block Tar hela bredden. Börjar på ny rad. <div>, <p>, <h1>
inline Tar bara den plats det behöver. Flöder med texten. Kan INTE ha width/height. <span>, <a>, <strong>
inline-block Flöder med texten MEN kan ha width/height. Bäst av båda världar. Knappar, taggar, badges.

Se skillnaden:

Block — varje element tar hela raden:

Jag är ett block-element
Jag också — ny rad!

Inline — flöder med texten: inline 1 inline 2 inline 3

Inline-block — flöder med texten men kan ha storlek:

Box A Box B Box C

Centrera innehåll — det klassiska tricket

.container {
  max-width: 800px;       /* Innehållet blir aldrig bredare än 800px */
  margin: 0 auto;         /* auto på vänster och höger = centrerat! */
}

/* "auto" säger åt webbläsaren: "fördela utrymmet jämnt"
   Resultat: lika mycket marginal på båda sidor = centrerat. */
Tips: Den här lektionens sektion använder just det tricket! max-width: 860px och margin-left: auto; margin-right: auto. Gör webbläsarfönstret smalare och bredare — se hur texten aldrig blir bredare än 860px men alltid förblir centrerad.

Uppdrag: Boxmodellen live

  1. Skapa en HTML-fil med en <div> som har width: 300px, padding: 20px och border: 5px solid red.
  2. Inspektera den med F12 och titta på "Box Model"-diagrammet — hur bred är den totalt?
  3. Lägg till box-sizing: border-box — titta igen. Vad ändrades?
  4. Centrera en <div> med max-width: 600px och margin: 0 auto.

Boss-level: Display-experiment

  1. Skapa tre <div>-element med bakgrundsfärg och text.
  2. De är block som standard — de ligger under varandra.
  3. Ändra till display: inline-block — vad händer?
  4. Ge dem width: 30% — nu har du en enkel tre-kolumnslayout!

Lektion 4: Praktiskt — Styla din HTML-sida

CSS-spaning — STEG 4: Den här sektionen använder allt vi lärt oss: box-sizing: border-box, centrerad layout med max-width, rgba()-bakgrunder på steg-rutorna, font-stacks, border-radius, padding, margin — hela paketet. Steg-rutorna nedan (.steg-ruta) har rgba(13, 181, 230, 0.08) som bakgrund och en subtil blå ram. Sammanfattningstabellen har rgba()-färg i rubrikraden. Det här är vad en "polerad" CSS-design ser ut som.

Nu kopplar du ihop allt. Du har en HTML-sida från Modul 1. Den har struktur, rubriker, listor, länkar, bilder. Men den ser fortfarande ut som ett raw-dokument. Dags att ge den stil.

Steg 1: Skapa en extern CSS-fil

Koppla CSS till HTML

Skapa en fil som heter style.css i samma mapp som din HTML-fil. Lägg sedan till den här raden i <head>:

<head>
  <meta charset="UTF-8">
  <title>Min sida</title>
  <link rel="stylesheet" href="style.css">
</head>

Det är allt. Nu kan du skriva CSS i style.css och den gäller för din HTML-sida.

Steg 2: Börja med grunden

Reset och bas-styling

Det första du gör i din CSS-fil — alltid:

/* Reset */
* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

/* Bas */
body {
  font-family: 'Segoe UI', system-ui, sans-serif;
  line-height: 1.6;
  color: #333;
  background-color: #f5f5f5;
  padding: 20px;
}

/* Centrera innehållet */
main {
  max-width: 800px;
  margin: 0 auto;
}

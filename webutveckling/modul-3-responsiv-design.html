Lektion 1: Introduktion till responsiv design och media queries

CSS-spaning — STEG 1: Här introducerar vi @media-queries! Den här presentationen har redan en media query som ändrar font-size på rubriker och padding på containern när skärmen är smalare än 768px. Gör fönstret smalare — ser du hur rubrikerna krymper? Det är en @media (max-width: 768px)-regel i aktion.

Varför responsiv design? Över 60% av all webbtrafik kommer från mobiler. Om din sida bara ser bra ut på en stor skärm missar du mer än hälften av dina besökare. Det är som att öppna en butik men låsa dörren för alla som inte kör bil dit.

Viewport-taggen — steg ett

Innan vi gör något annat behöver vi en viktig rad i <head>:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Utan den här raden kommer mobilen att låtsas att den är en desktop-skärm och zooma ut allt till en miniversion. Med den säger vi: "Hej mobil, din bredd ÄR din bredd — visa sidan i rätt storlek."

Kolla själv: Den här sidan har viewport-taggen. Öppna den på din mobil — texten ska vara läsbar utan att zooma. Om du tar bort viewport-taggen blir allt pyttelitet.

Media queries — CSS som lyssnar på skärmen

En media query är en CSS-regel som bara gäller om ett villkor är sant. Tänk det som en if-sats för CSS:

/* Vanlig CSS — gäller alltid */
h1 {
  font-size: 2em;
}

/* Media query — gäller BARA om skärmen är 768px eller smalare */
@media (max-width: 768px) {
  h1 {
    font-size: 1.4em;
  }
}

/* Du kan ha flera breakpoints */
@media (max-width: 480px) {
  h1 {
    font-size: 1.2em;
  }
}

Vanliga breakpoints

EnhetBreddTypisk breakpoint

OBS: Det finns inga "rätta" breakpoints. Välj breakpoints baserat på din design, inte på specifika enheter. Om layouten ser konstig ut vid 750px — lägg en breakpoint där!

Mobile-first vs desktop-first

Det finns två sätt att tänka:

/* DESKTOP-FIRST: börja med desktop, anpassa neråt */
/* "Normalt" = desktop-layout */
.container { width: 900px; }

@media (max-width: 768px) {
  .container { width: 100%; }  /* Skärmen är liten — ta hela bredden */
}

/* ─────────────────────────────────── */

/* MOBILE-FIRST: börja med mobil, anpassa uppåt */
/* "Normalt" = mobil-layout */
.container { width: 100%; }

@media (min-width: 768px) {
  .container { width: 900px; }  /* Skärmen är stor — begränsa bredden */
}

Mobile-first anses vara bäst idag. Du börjar med den enklaste layouten (mobil) och lägger till komplexitet för större skärmar. Det tvingar dig att tänka: "Vad är viktigast?" — och det ger oftast bättre resultat.

Uppdrag: Din första media query

  1. Skapa en HTML-sida med en rubrik, tre paragrafer och en bakgrundsfärg.
  2. Lägg till en @media (max-width: 600px) som ändrar bakgrundsfärgen till en annan färg.
  3. Lägg till en till media query som ändrar fontstorleken vid 480px.
  4. Dra i webbläsarfönstret och se färgen skifta — magi!

Utmaning: Breakpoint-detektiv

  1. Öppna Chrome DevTools (F12) och klicka på enhetsikonen (Ctrl+Shift+M).
  2. Välj "Responsive" och dra i bredden.
  3. Besök tre sajter du gillar — vid vilka bredder ändras layouten? Skriv ner breakpoints!

Lektion 2: Flexbox — introduktion och responsiv layout

CSS-spaning — STEG 2: Nu använder presentationen display: flex på riktigt! Navigationen i toppen är en flexbox-lista med flex-wrap: wrap och gap: 8px. Demo-rutorna nedan använder flex: 1 1 200px — de fyller raden och radbryts automatiskt. Det här är kraften med flexbox: en rad CSS ändrar allt.

Flexbox är det bästa som hänt CSS. Innan flexbox var det en mardröm att centrera saker, lägga element bredvid varandra eller fördela utrymme jämnt. Med flexbox? En rad: display: flex.

Grundkonceptet: container och items

Flexbox har alltid två delar:

  • Flex container — föräldern som har display: flex
  • Flex items — barnen som automatiskt placeras i rad
<div class="container">       <!-- FLEX CONTAINER -->
  <div class="item">A</div>  <!-- FLEX ITEM -->
  <div class="item">B</div>  <!-- FLEX ITEM -->
  <div class="item">C</div>  <!-- FLEX ITEM -->
</div>

<style>
.container {
  display: flex;   /* En rad — och barnen lägger sig i rad! */
}
</style>

flex-direction: rad eller kolumn

.container {
  display: flex;
  flex-direction: row;      /* Standard — horisontell rad (→) */
  flex-direction: column;   /* Vertikal kolumn (↓) */
}

justify-content: fördela utrymme längs huvudaxeln (horisontellt som standard)

VärdeVad det gör
flex-startAlla items i början (standard)
centerAlla items i mitten
flex-endAlla items i slutet
space-betweenJämnt fördelat, inget utrymme i kanterna
space-aroundJämnt fördelat, halva utrymmet i kanterna
space-evenlyExakt lika mycket utrymme överallt

align-items: justera längs tväraxeln (vertikalt som standard)

VärdeVad det gör
stretchItems sträcker sig till containerns höjd (standard)

flex-wrap: radbrytning

Utan flex-wrap tvingas alla items in på en enda rad — de krymper istället för att radbrytas. Med flex-wrap: wrap hoppar de ner till nästa rad om de inte får plats. Det här är nyckeln till responsivitet!

.container {
  display: flex;
  flex-wrap: wrap;     /* Items radbryter vid behov */
  gap: 16px;           /* Avstånd mellan items */
}

.item {
  flex: 1 1 250px;     /* Minst 250px bred, men växer om det finns plats */
}

/* flex: 1 1 250px är shorthand för:
   flex-grow: 1     — item får växa
   flex-shrink: 1   — item får krympa
   flex-basis: 250px — utgångsstorlek */

Live-demo: Flexbox i aktion

De här rutorna nedan använder display: flex med flex-wrap: wrap. Gör fönstret smalare — de hoppar ner under varandra!

display: flex

Lägger barnen i en rad (eller kolumn). En rad som ändrar allt.

flex-wrap: wrap

Låter items radbrytas istället för att krympa till ingenting.

gap: 16px

Avstånd mellan items. Mycket enklare än margin-hack!

Det magiska tricket: flex: 1 1 250px på items + flex-wrap: wrap på containern = en responsiv layout utan media queries. Items fyller raden, och när de inte får plats radbryts de. Prova det ovan!

Uppdrag: Flexbox-kort

  1. Skapa en sida med en container-div och 4 kort-divar inuti.
  2. Ge containern display: flex, flex-wrap: wrap och gap: 16px.
  3. Ge varje kort flex: 1 1 200px, en bakgrundsfärg och padding.
  4. Gör fönstret smalare — korten ska radbrytas automatiskt!

Utmaning: Centrera allt

  1. Skapa en div med height: 100vh (hela skärmens höjd).
  2. Använd display: flex, justify-content: center och align-items: center.
  3. Lägg en rubrik inuti — den ska vara perfekt centrerad. Det som brukade kräva 10 rader CSS tar nu 3.

Lektion 3: Praktiskt — Anpassa webbplatsen

CSS-spaning — STEG 3: Nu kombineras flexbox och media queries! Navigationen i toppen har en @media (max-width: 768px) som byter flex-direction till column. Korten nedan har flex: 1 1 250px normalt men flex: 1 1 100% vid 768px. Gör fönstret smalare och se hur allt anpassar sig — det är exakt vad du ska lära dig bygga.

Nu sätter vi ihop allt. Media queries och flexbox var för sig är bra. Tillsammans är de oslagbara. I den här lektionen bygger vi responsiva mönster som du kommer använda om och om igen.

Mönster 1: Responsiv navigation

De flesta sajter har en horisontell meny på desktop som blir vertikal (eller en hamburgermeny) på mobil. Så här gör du den enklaste versionen:

<nav>
  <ul class="nav-meny">
    <li><a href="#">Hem</a></li>
    <li><a href="#">Om oss</a></li>
    <li><a href="#">Kontakt</a></li>
  </ul>
</nav>

<style>
.nav-meny {
  display: flex;
  list-style: none;
  gap: 16px;
  padding: 0;
}

/* På små skärmar: byt till kolumn */
@media (max-width: 600px) {
  .nav-meny {
    flex-direction: column;
  }
}
</style>

Titta på navigationen i toppen av den här sidan — den gör exakt det här!

Mönster 2: Kort-layout som anpassar sig

Tre kort i rad på desktop, en i taget på mobil. Det här mönstret ser du överallt — bloggar, nyheter, produktsidor:

.kort-container {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}

.kort {
  flex: 1 1 300px;   /* Minst 300px — radbryts automatiskt */
  padding: 20px;
  background: #16213e;
  border-radius: 8px;
}

/* Eller med media query för mer kontroll: */
@media (max-width: 768px) {
  .kort {
    flex: 1 1 100%;  /* Ta hela bredden på tablet och mobil */
  }
}

Mönster 3: Bilder som skalar

img {
  max-width: 100%;    /* Aldrig bredare än sin container */
  height: auto;       /* Behåll proportionerna */
  display: block;     /* Tar bort extra utrymme under bilden */
}

De här tre raderna borde finnas på varje sida du bygger. De ser till att bilder aldrig "spränger" layouten på små skärmar.

Chrome DevTools: din bästa kompis

Du behöver inte en mobil för att testa responsivitet. Chrome har ett inbyggt verktyg:

  1. Öppna DevTools: Ctrl+Shift+I (eller F12)
  2. Klicka på enhetsikonen (eller tryck Ctrl+Shift+M)
  3. Välj "Responsive" i dropdown-menyn
  4. Dra i bredden — se hur sidan reagerar
  5. Testa specifika enheter: iPhone, iPad, Galaxy...
Proffs-tips: Håll DevTools öppna medan du kodar CSS. Ändra bredden fram och tillbaka. Om layouten ser konstig ut vid någon bredd — lägg till en media query just där. Designa för din design, inte för specifika enheter.

Uppdrag: Responsiv profilsida

  1. Skapa en sida med en <header> (med ditt namn), en <nav> (med 3–4 länkar) och en <main> med 3 kort (intressen, skola, kontakt).
  2. Använd flexbox för navigationen och korten.
  3. Lägg till en media query vid 768px som gör navigationen vertikal och korten fulla bredden.
  4. Testa i DevTools med minst 3 olika skärmstorlekar.

Boss-level: Responsiv bildgalleri

  1. Hitta 6 bilder (t.ex. från Unsplash).
  2. Skapa ett galleri med flexbox: 3 kolumner på desktop, 2 på tablet, 1 på mobil.
  3. Alla bilder ska ha max-width: 100% och bra alt-text.

Lektion 4: Fördjupning i responsiv design

CSS-spaning — STEG 4: Den här sektionens enhets-rutor använder display: grid med grid-template-columns: repeat(auto-fit, minmax(220px, 1fr)) — det är CSS Grid! Padding och margins använder nu rem istället för px. Ändra webbläsarens standard-fontstorlek (Inställningar → Utseende) och se hur rem-baserade värden skalar med.

Pixlar är inte alltid svaret. Hittills har vi mest använt px — fasta pixelvärden. Men responsiv design handlar om att saker ska vara flexibla. Dags att lära sig fler enheter.

CSS-enheter: px vs em vs rem vs % vs vw/vh

px — pixlar

Fast storlek. 16px är alltid 16px. Bra för borders och skuggor. Dåligt för fontstorlekar om du vill att sidan ska vara tillgänglig.

em — relativt till föräldern (eller elementet självt)

1em = förälderns fontstorlek (för font-size) eller elementets egen fontstorlek (för padding, margin m.m.). Om föräldern har 16px är 1.5em = 24px. Kan bli förvirrande med nästlade element!

rem — relativt till root

1rem = html-elementets fontstorlek (oftast 16px). Konsekvent och förutsägbar. Använd rem för fontstorlekar!

% — procent

Relativt till föräldern. width: 50% = halva förälderns bredd. Perfekt för flexibla layouter.

vw / vh — viewport

1vw = 1% av fönstrets bredd. 1vh = 1% av fönstrets höjd. Bra för hero-sektioner: height: 100vh.

min-width, max-width, min-height

De här egenskaperna sätter gränser. De är guld värda för fluid design:

.container {
  width: 90%;         /* Flexibel: 90% av föräldern */
  max-width: 900px;   /* Men aldrig bredare än 900px */
  min-width: 300px;   /* Och aldrig smalare än 300px */
  margin: 0 auto;     /* Centrerad */
}

img {
  width: 100%;
  max-width: 600px;   /* Bilden växer med containern, men max 600px */
  min-height: 200px;  /* Alltid minst 200px hög */
}

CSS Grid — en sneak peek

Flexbox är bra för en-dimensionella layouter (en rad eller en kolumn). Men ibland vill du ha ett riktigt rutnät — rader OCH kolumner. Då finns CSS Grid:

.grid-container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);  /* 3 lika breda kolumner */
  gap: 16px;
}

/* Responsiv grid med auto-fit: */
.grid-container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 16px;
}
/* Kolumnerna anpassar sig automatiskt efter tillgänglig plats! */
Flexbox vs Grid? Tumregel: Flexbox för en dimension (en meny, en rad knappar). Grid för två dimensioner (hela sidlayouter, bildgallerier). Du kan (och bör!) använda båda i samma projekt. Vi kommer djupdyka i Grid i en senare modul.

Responsiva bilder med object-fit

img {
  width: 100%;
  height: 300px;
  object-fit: cover;   /* Fyller utrymmet utan att förvrängas */
}

/* object-fit-värden:
   cover    — fyller hela ytan, klipper om det behövs
   contain  — hela bilden syns, men kan få tomrum
   fill     — sträcker bilden (förvränger!)
   none     — originalstorlek
*/

object-fit: cover är perfekt för kort-layouts där alla bilder ska vara lika höga men kan ha olika proportioner.

Uppdrag: Enhetsexperiment

  1. Skapa en sida med en rubrik i rem, en paragraf i em och en container i %.
  2. Ändra webbläsarens standardfontstorlek (Chrome → Inställningar → Utseende → Teckenstorlek). Vad händer med rem-värdena? Med px-värdena?
  3. Skapa en div med height: 100vh och en bakgrundsfärg. Vad händer när du skrollar?

Boss-level: Grid-galleri

  1. Skapa ett bildgalleri med display: grid och grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)).
  2. Ge alla bilder object-fit: cover och en fast höjd.
  3. Galleriets kolumner ska anpassas automatiskt — ingen media query behövs!

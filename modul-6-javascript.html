Lektion 1: Vad är JavaScript?

CSS-spaning — NY: cursor: pointer och user-select
Ser du att muspekaren ändrar form när du hovrar över knapparna och demo-elementen nedan? Det är cursor: pointer — den lilla handen som talar om "det här är klickbart". Utan den ser knappar och klickbara element likadana ut som vanlig text, vilket är förvirrande. user-select: none förhindrar att text av misstag markeras när man klickar snabbt, och user-select: all markerar hela kodblock med ett enda klick.

Okej, du vet redan att webben har tre lager. HTML är skelettet, CSS är kläderna. JavaScript är det som gör att kläder kan bytas, skelett kan röra sig och saker faktiskt händer när du klickar. Utan JS är en webbsida en affisch. Med JS är det ett program.

De tre pelarna — en snabb påminnelse

SpråkVad det görExempel
HTML Struktur — vad som finns "Det här är en rubrik och en knapp."
CSS Utseende — hur det ser ut "Knappen är röd och har rundade hörn."
JavaScript Beteende — vad som händer "När man klickar på knappen ändras texten."

Lite historia (snabbversion)

JavaScript skapades 1995 av Brendan Eich på Netscape. Han fick uppdraget att bygga ett skriptspråk för webben. Han var klar på 10 dagar. (Seriöst. Tio dagar. Det förklarar en del konstigheter i språket.)

Det hette ursprungligen Mocha, sedan LiveScript, och döptes till JavaScript av marknadsskäl — Java var populärt just då. Trots likheten i namn har JavaScript och Java ungefär lika mycket gemensamt som en hamburgare och ett hamburger-emoji.

Idag är JavaScript det mest använda programmeringsspråket i världen. Det körs i din webbläsare, på servrar, i mobilappar och i smarta kylskåp. Det man byggde ihop på 10 dagar driver en stor del av internet.

Var körs JavaScript?

Du behöver inte installera någonting. JS körs direkt i webbläsaren — och du har faktiskt en JS-konsol tillgänglig just nu:

Prova nu: Tryck F12 (eller högerklicka → Inspektera) och klicka på fliken Console. Skriv 1 + 1 och tryck Enter. Skriv sedan alert("hej") och tryck Enter. Grattis — du har kört JavaScript.

Tre sätt att lägga till JS på en sida

<!-- 1. Inline i HTML-taggen (undvik — svårt att läsa) -->
<button onclick="alert('hej')">Klicka</button>

<!-- 2. Script-block i HTML-filen (bra för nybörjare) -->
<script>
  console.log("hej från script-blocket!");
</script>

<!-- 3. Extern fil (standard i riktiga projekt) -->
<script src="script.js"></script>

Vi börjar med script-block — det är enklast att förstå och se sambandet med HTML. I riktiga projekt delar man alltid upp i en separat .js-fil.

console.log() — din bästa vän

console.log() skriver ut saker i konsolen. Det är ditt viktigaste felsökningsverktyg. Proffsutvecklare använder det hela tiden. Det kostar ingenting och det stör ingen — det syns bara i DevTools.

console.log("Hej!");                    // Skriver ut: Hej!
console.log(42);                        // Skriver ut: 42
console.log("Summan är:", 3 + 4);       // Skriver ut: Summan är: 7
console.log(true);                      // Skriver ut: true

Variabler: let och const

En variabel är en namngiven låda där du lagrar ett värde. Du kan läsa värdet, byta det och använda det i beräkningar.

// let — ett värde som kan ändras
let poang = 0;
poang = poang + 10;  // nu är poang 10
poang = 100;         // nu är poang 100

// const — ett värde som INTE kan ändras
const spelarnamn = "GamerXL99";
// spelarnamn = "nåt annat";  <-- detta kraschar!

// Använd const om du vet att värdet inte ska ändras.
// Använd let om värdet kommer att ändras.
// Använd ALDRIG var — det är gammaldags och har konstiga regler.

Datatyper — tre du behöver nu

TypVad det ärExempel
string Text — alltid inom citattecken (enkla eller dubbla). "hej", 'GTA VI', `Hej ${namn}`
number Tal — utan citattecken. Heltals och decimaltal. 42, 3.14, -7
boolean Sant eller falskt — och bara det. Inget citattecken. true, false 
const namn    = "Alex";            // string
const alder   = 17;                // number
const inloggad = true;             // boolean

// Kolla typen med typeof:
console.log(typeof namn);         // "string"
console.log(typeof alder);        // "number"
console.log(typeof inloggad);     // "boolean"

// Strängsammansättning med template literals (backtick):
console.log(`Hej ${namn}, du är ${alder} år!`);
// Skriver ut: Hej Alex, du är 17 år!

Lektion 2: Enkel interaktivitet — knapp som gör saker

CSS-spaning — NY: transition
Den här lektionen introducerar transition — det som gör att CSS-ändringar inte sker direkt utan glider mjukt. Utan transition: en knapptryckning byter bakgrundsfärg momentant. Med transition: background-color 0.4s ease glider färgen över 0.4 sekunder. Det är viktigt just nu för att JS kan ändra CSS-värden direkt — och med transition blir det visuellt tydligt vad som händer. Kolla på demo-rutorna nedan!

Nu börjar det roliga på riktigt. Vi ska hitta HTML-element med JS, ändra dem och reagera på klick. Det är egentligen tre saker som måste sitta: hitta elementet, lyssna på klicket, göra förändringen.

alert() — enkel men irriterande

alert() visar en popup-ruta. Det är det enklaste sättet att bevisa att JS körs. Det är också det mest irriterande. Ingen riktigt webbplats använder den — men den är bra för att förstå grunderna.

alert("Hej!");               // Popup med text
prompt("Vad heter du?");     // Popup där användaren kan skriva
confirm("Är du säker?");     // Popup med OK / Avbryt

document.querySelector() — hitta element

Det är här JS och HTML kopplas ihop. document.querySelector() letar igenom sidan och returnerar det första elementet som matchar en CSS-selektor — precis som CSS-selektorer fast i JS.

// Hitta ett element med ID:
const rubrik = document.querySelector('#min-rubrik');

// Hitta ett element med klass:
const knapp = document.querySelector('.demo-knapp');

// Hitta ett element med taggnamn:
const forstaP = document.querySelector('p');

// Hitta ALLA matchande element (ger en lista):
const allaKnappar = document.querySelectorAll('button');

.textContent — ändra text

När du hittat ett element kan du läsa eller skriva dess textinnehåll med .textContent.

Den här texten kan ändras av JavaScript!

// Läs textinnehållet:
const rubrik = document.querySelector('#text-demo');
console.log(rubrik.textContent);  // Skriver ut texten

// Skriv nytt textinnehåll:
rubrik.textContent = 'Ny text!';

.style — ändra CSS från JavaScript

Du kan ändra CSS-egenskaper direkt med JS via .style. Observera att bindestreck i CSS blir camelCase i JS: background-color → backgroundColor.

Den här rutan ändrar utseende när du klickar!

const ruta = document.querySelector('#min-ruta');

// CSS-egenskap background-color → JS: backgroundColor
ruta.style.backgroundColor = '#e94560';

// CSS-egenskap font-size → JS: fontSize
ruta.style.fontSize = '2em';

// CSS-egenskap border-radius → JS: borderRadius
ruta.style.borderRadius = '50%';

Events — reagera på klick

En event (händelse) är något som händer på sidan: ett klick, en tangenttryckning, musen rör sig. Du kan lyssna på events och köra kod när de sker.

<!-- Sätt 1: onclick direkt i HTML (snabbt men lite stökigt) -->
<button onclick="alert('klickad!')">Klicka</button>

<!-- Sätt 2: addEventListener (bättre — separerar HTML och JS) -->
<button id="min-knapp">Klicka</button>

<script>
  const knapp = document.querySelector('#min-knapp');

  knapp.addEventListener('click', function() {
    alert('klickad!');
  });
</script>
Varför addEventListener? Med onclick i HTML blandar du HTML och JS — svårare att underhålla. Med addEventListener kan du lyssna på massor av events (klick, hover, tangentbord) utan att röra HTML-koden. Det är standardsättet i riktiga projekt.

Antal klick:

Lektion 3: Praktiskt — toggle, FAQ och räknare

CSS-spaning — NY: CSS-klasser designade för JS
Det här är ett nyckelmönster: du definierar klasser i CSS som "sätter på" ett beteende, och sedan låter JS lägga till och ta bort dem. .hidden { display: none; } gör ett element osynligt. .aktiv { ... } ger en aktiv stil. .ljust-lage { background-color: #f0f4ff; } byter tema. JS ändrar aldrig CSS direkt — den togglar bara klasser. Det gör det lätt att underhålla: ändra utseendet i CSS-filen, JS-koden behöver inte röras. Se classList.toggle() nedan.

Nu kombinerar vi allt. Det här är de vanligaste mönstren du ser på riktiga sidor: dark mode, accordion, räknare, bildspel. Alla bygger på exakt samma grundidé: en händelse → ändra DOM eller klasser.

classList — lägg till, ta bort och toggle klasser

const box = document.querySelector('#min-box');

// Lägg till en klass:
box.classList.add('aktiv');

// Ta bort en klass:
box.classList.remove('aktiv');

// Toggle: lägg till om den saknas, ta bort om den finns:
box.classList.toggle('aktiv');

// Kolla om elementet har en klass:
if (box.classList.contains('aktiv')) {
  console.log('aktiv!');
}

Demo 1: Dark / Light mode

Det här är en demo-yta. Klicka på knappen för att byta mellan mörkt och ljust läge. Lägg märke till att övergången är mjuk — det är transition i CSS som gör det!

const yta = document.querySelector('#mode-demo');
const knapp = document.querySelector('#mode-toggle-knapp');

knapp.addEventListener('click', function() {
  yta.classList.toggle('ljust-lage');

  if (yta.classList.contains('ljust-lage')) {
    knapp.textContent = 'Byt till mörkt läge';
  } else {
    knapp.textContent = 'Byt till ljust läge';
  }
});

Demo 2: FAQ-accordion — visa/dölj innehåll

En accordion döljer allt från start och visar ett svar i taget. Det klassiska mönstret på alla FAQ-sidor.

Python körs på servern, inte i webbläsaren. Webbläsare förstår bara JavaScript — det är det enda skriptspråk som är inbyggt i varje webbläsare. Därför HAR man liksom valt JS för webben, inte valt bort Python.
Nej. Absolut inte. De delar namn av historiska marknadsföringsskäl från 1995. JavaScript och Java är ungefär lika lika som "hamburgare" och "hamster".
Ja. Frontend-utveckling utan JavaScript är som att laga mat utan spis — tekniskt möjligt men extremt begränsat. JS är numera obligatoriskt för i princip alla webbrelaterade jobb.
Nytt här: querySelectorAll väljer ALLA element som matchar (inte bara det första). forEach loopar igenom alla och gör något med varje. 
// Hämta ALLA .faq-fraga-element (en lista)
const faqFragor = document.querySelectorAll('.faq-fraga');

// Loopa igenom dem och lägg till klicklyssnare på varje
faqFragor.forEach(function(fraga) {
  fraga.addEventListener('click', function() {
    const faqItem = fraga.parentElement;   // gå upp ett steg i DOM
    faqItem.classList.toggle('aktiv');     // sätt på/av klassen
  });
});

Lektion 4: Sammanfattning och vad händer härnäst

CSS-spaning — NY: @keyframes animationer
Det här är en teaser på vad CSS kan göra på egen hand — utan JS. @keyframes definierar en animation steg för steg. Du ger den ett namn, beskriver hur den börjar och slutar (och eventuella mellansteg), och sedan kopplar du den till ett element med animation: namn duration timing iteration. Texten "REGNBÅGE" nedan animeras med rena CSS-keyframes. Korten som glider in använder animation-delay för att starta lite efter varandra. Det kräver noll JavaScript.

Du har faktiskt lärt dig ganska mycket. Det känns kanske inte så — men tänk på vad du kunde i förra veckan kontra nu. Låt oss gå igenom det ordentligt.

KonceptVad det ärExempel

string, number, boolean De tre vanligaste datatyperna. "hej", 42, true
console.log() Skriv ut i konsolen — felsökningsverktyg. console.log(poang);
document.querySelector() Hitta ett HTML-element med CSS-selektor. document.querySelector('#knapp')
.textContent Läs eller skriv text i ett element. el.textContent = 'ny text';
.style Ändra CSS direkt från JS. el.style.color = 'red';
addEventListener('click', ...) Lyssna på en händelse och kör kod. knapp.addEventListener('click', fn)

Debugging — vad gör du när det inte funkar?

JS slutar tyst om det är ett fel — ingenting händer och du undrar varför. Stegen för att hitta felet:

  1. F12 → Console. Finns det ett rött felmeddelande? Läs det — det säger oftast exakt vad som är fel och på vilken rad.
  2. Vanliga fel:
    • Cannot read properties of null — querySelector hittade inget. Kolla att id/class stämmer.
    • is not a function — du har stavat fel på en metod.
    • Uncaught ReferenceError — du använder en variabel som inte är definierad.
  3. Lägg till console.log() överallt tills du hittar var det går fel.
  4. Googla felmeddelandet. Nästan alla fel har svar på Stack Overflow.
Tips: Skriv console.log("här är vi nu") på strategiska ställen i koden. Om den skriver ut nåt vet du att koden körts. Om den inte skriver ut nåt har koden inte körts — och nu vet du var felet är.

@keyframes regnbage {
  0%   { color: #e94560; }
  25%  { color: #f5c518; }

/* Koppla till ett element: */
.regnbage-text {
  animation: regnbage 3s linear infinite;
}

/* Glida in från nedanför: */
@keyframes glid-in {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

.animerat-kort {
  animation: glid-in 0.6s ease both;
}
